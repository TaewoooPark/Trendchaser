# Trendchaser 2026-05-04 update

**TL;DR — briefs now show "signals published yesterday" far more often. The slot that GitHub trending used to dominate is now filled by first-party sources: lab releases, model uploads, and breaking news.**

The direction of this update is simple. We optimized for capturing **what was just announced** rather than **what's already been viral**. To get there, we added new channels, refined the scoring formula, and structurally prevented stale trending items from taking over the top of the brief.

## What's been added

16 new channels and additional coverage across 31 foundation labs. The net effect: the chance that the first item in your brief was published less than 24 hours ago is now substantially higher.

### New path 1 — GitHub release firehose (14 added)

When a model or framework gets a new version tag on GitHub, the fastest-updating signal is the repo's `releases.atom` feed. This revision adds LLM infrastructure, agent frameworks, and MCP SDKs in one go:

| Category | Channels |
|---|---|
| OpenAI ecosystem | openai-python, openai-agents-python, openai/swarm |
| HuggingFace stack | transformers, accelerate, diffusers, peft |
| Inference engines | vllm, ollama, llama.cpp |
| Agent frameworks | langchain, autogen |
| MCP SDKs | python-sdk, typescript-sdk |

Combined with the existing `claude_code_releases` and `mcp_releases`, the **release channel count is now 16**. Thanks to weights of 1.5–1.6, the moment vllm cuts a new version or transformers merges a new model family, that change surfaces in the very next brief.

### New path 2 — Tracking fresh HuggingFace uploads

The existing `hf_models_trending` covers "models that are already popular." That means models freshly pushed by a lab don't reliably make it through.

The new `hf_models_new` walks the **47 whitelisted labs** (meta-llama, google-deepmind, nvidia, mistralai, deepseek-ai, Qwen — plus the 31 added in this revision) one by one, collecting anything uploaded within the past 7 days. In practice:

- Yesterday Qwen uploaded a new 7B model → candidate for the next slot's brief
- Last week BlackForestLabs released a Flux successor → still inside the 7-day window, also a candidate

The existing trending channel was lowered from weight 1.3 to 0.9, cleanly separating the two roles.

### New path 3 — Hacker News breaking

The existing `hn_top` ranks by cumulative score — it surfaces stories that **already** went viral. The new `hn_breaking` watches **the past 4 hours** and pulls anything matching AI keywords that has crossed 75 points.

```
hn_top      = "what's on the HN front page right now"
hn_breaking = "what's about to be on the front page"
```

The two channels cover different time windows and complement each other.

## Scoring change — introducing the novelty axis

A **novelty** axis was added to the existing five, expanding the formula to six axes.

| Axis | Before | After |
|---|---|---|
| signal | 30% | 25% |
| affinity | 30% | 25% |
| recency | 25% | 20% |
| **novelty** | — | **20% (new)** |
| velocity | 10% | 5% |
| freshness | 5% | 5% |

Novelty scores are assigned as follows:

| Source type | Novelty score |
|---|---|
| First-party source (lab blog, release, papers) | **100** |
| Curator / aggregator | 65 |
| Trending (created within 7 days) | 80 |
| Trending (created within 30 days) | 50 |
| Trending (created within 90 days) | 25 |
| Trending (older than 90 days) | 0 |
| arXiv raw | 50 |

This axis quantitatively distinguishes "a 2-year-old repo riding a one-off hype wave at the top of GitHub trending" from "something nvidia released minutes ago." Even among trending items, the fresher one ends up higher in the brief.

---

## Hard cap on trending

A new hard cap is now enforced at the code level:

- **AI section**: max 2 trending sources
- **General section**: max 1 trending source

This structurally prevents the "trending takes up half the brief" pattern. On top of that, a freshness multiplier (≤7d 1.3× / >90d 0.4×) is applied during enrichment, so old trending repos don't even get allocated body-extraction resources.
