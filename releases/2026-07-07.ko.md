## PRISMA-Codex 전환

기존 운영은 Claude Code Routines의 managed routine에 가까운 형태였습니다. 이번 전환 이후에는 GitHub Actions가 정시 실행과 저장소 checkout/publish를 담당하고, Codex auth가 scoring·editorial review·brief 작성 루프를 담당합니다.

전환으로 얻은 장점:

- Codex 기반 실행과 GitHub Actions 상태가 한 저장소 흐름 안에 남아 디버깅이 쉬워짐
- private repo의 Actions 제약을 고려한 workflow 권한·secret 구조 명확화
- KST 슬롯 계산과 boundary wait를 코드로 강제해 18시 루틴이 조기 실행되는 문제를 방지
- `CODEX_ACCESS_TOKEN` 또는 `CODEX_AUTH_JSON` 누락 시 조용히 약한 브리프를 보내지 않고 명시 실패
- Telegram 마지막 생성 footer에 `by Codex`를 넣어 운영 주체를 식별 가능

## 새 학술·연구 계층

이번 확장의 핵심은 “ICML만 체리피킹”이 아니라, 연구 트렌드 계층 전체를 넓히는 것입니다.

- `arxiv_llm_agents`, `arxiv_multimodal_video`, `arxiv_robotics_embodied`: raw arXiv보다 좁고 현재 트렌드에 가까운 focused query
- `semantic_scholar_latest_ai`, `openalex_latest_ai`: 독립 학술 index를 통한 교차 확인
- `paperswithcode_ai`: 논문과 구현/adoption이 붙은 신호
- `icml_2026_schedule`, `neurips_2026_schedule`, `iclr_2026_schedule`, `mlsys_2026_schedule`: 최상위 학회 schedule/program surface
- `acl_2026_proceedings`, `cvpr_2026_proceedings`: proceedings 기반 발견 경로
- `watch_openreview_*`: OpenReview API가 challenge/403으로 흔들릴 때 venue page HTML watch로 보완

## 가중치 정책

확장된 소스는 기존 소스와 같은 층에 무작정 넣지 않고 권위·신선도·노이즈를 기준으로 나눴습니다.

| 레이어 | 대략적 weight | 의도 |
|---|---:|---|
| 최상위 공식/학회 schedule | 1.55-1.75 | ICML/NeurIPS/ICLR/MLSys 등 신뢰도 높은 1차 신호 |
| 공식 docs/watch | 1.25-1.35 | 모델 문서·release note·leaderboard diff |
| 구현/adoption index | 1.10-1.15 | Papers With Code, GitHub Search 등 실사용 움직임 |
| 학술 index | 1.05 | Semantic Scholar/OpenAlex. 넓게 보되 단독 권위는 낮춤 |
| 커뮤니티 smoke | 0.6-1.1 | Reddit/Bluesky/Mastodon 등 발견용. 교차 확인 전 단독 승격 억제 |

## 교차 소스 확인 정책

동일 entity가 서로 다른 소스에서 잡히면 `cross_confirm_count`, `cross_confirm_sources`, `cross_confirm_key`를 부여합니다. key는 DOI, arXiv ID, OpenReview ID, GitHub repo, Hugging Face model ID, 모델/릴리즈명, 정규화 title fallback을 사용합니다.

중요한 점은 이 정책이 “방어를 푸는 정책”이 아니라 “후보 순위를 올리는 정책”이라는 것입니다. 두 곳 이상에서 보였더라도 다음 조건은 그대로 통과해야 합니다.

- 원본 URL이 fetcher가 실제 수집한 URL이어야 함
- 14일 brief history와 URL/title/entity 중복이 없어야 함
- slot window와 freshness gate를 통과해야 함
- load-bearing claim은 원문 확인 또는 보수적 표현으로 처리되어야 함

## 검증 상태

PRISMA-Codex 구현 레포에서 다음 검증을 통과했습니다.

- `bash tests/run_all.sh`
- `npm --prefix prisma-proxy run typecheck`
- `git diff --check`
- dry-run 기준: `items=290`, `active_sources=27`, `failures=0`

로컬 환경에는 `~/Desktop/forcodex/.env`가 없어 Vercel X/watch 실서비스 probe는 건너뛰었지만, GitHub Actions secret/variable 구조에 맞춰 경로는 연결했습니다.
